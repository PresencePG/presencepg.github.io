What the client asked for:

  • Small & Fast

  • Customizable

  • Well Documented (for further dev work)

Here is where we intoduce the clients asked for as requirments for the types of graphs. It's immportant to emphasize that interpiting this is important in selecting something that will grow with the project.

The options:

  • All Inclusize Chart Library (i.e. Goolge Charts)

  • Boutique Graphs (rickshow.js, HighStock)

  • DOM Manipulation (D3, Raphael)

Lots of options for creating data vis. But three basic Catagoies prevail

Preconfigured, Specialized (Boutique) and scaffolding (clay). The scaffolding is simply a tool for painting and moving shapes

Why D3:

  • The charts and visulzations would be extended.

  • Clear requirment for unique visulazations

Here we need to emphasize that customization is the achilles heal of pre-configured graphing libraries

JavaScript Library for DOM manipulation.

Data Driven Documents.

Redefine what D3 is. Not a charting library but a way to present data in a web browser. Each visualzation is bespoke (with lots of good patterns to copy from). The code and charts is as good as you make it.

The tech side d3 satisfies:

  • Responsize: Tablet and phone resultion supported

  • Fast (i.e. 10+ small sized graphs)

  • Custom anything (labels, units and interactions)

  • A solid Community (Lots of examples and snippets to borrow from)

Technical Criteria we needed to support?

  1. Customization
  2. It needs to play nice with Angular
  3. Nexus and iPhone
  4. New visulaization would be added to the platform
  5. Scale and Range
